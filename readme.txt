Hello there guys! 

The plan (for now) is to make an isolation-chess-style game. Ref: https://scratch.mit.edu/projects/241565591/

For a cursory glance at the scope of the project, the project should ideally end up with: 

1. clear, OOP-oriented scalable architecture
2. proper usage of the curses library (if you use Windows the best option is to use WSL)
3. all features that exist in the original game (ideally both cloud & CPU gameplay)
4. properly designed UI that uses ASCII characters to mimic video game interface (not super high standards. Something like https://github.com/Meteony/Project-OLED-1330 should be good enough. Do it in a text editor first)
5. figure out ways to implement all other required features (usage of rand(), file i/o - probably something someting i/ofstream, dynamic memory allocation, etc.)
...

AAAAnd that should be it. Have fun and work hard. The workload should be managable as long as we divide the task early on and set proper deadlines. The key is to cooperate.
